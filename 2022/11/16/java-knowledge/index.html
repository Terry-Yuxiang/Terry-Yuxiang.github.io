Proxy与Invocationhandler

Proxy实现动态代理的三个参数

  1. classloader: boot, ext, app
    作用：实现类加载
    使用: 使用线程加载Thread.currentThread().getContxtClassloader()，或者使用当前类加载Test.class.getClassloader()都可以。
    使用线程这种方式相对来说更稳妥一些。
  2. CLass[] interfaceClass (这里要注意，jdk提供的proxy方式只支持接口.实际动态代理的实现有两种。1:实现接口 2: 实现父亲类)
  3. Invocationhandler 接口，自定义实现类
    在动态代理里，动态代理都会带哦用handler里的invoke方法，invoke方法再决定掉用t1,t2,t3。
    invoke中也有三个参数:
  4. proxy 代理对象。
  5. Method对象，记录使用者调用的代理方法，未来可以基于这个方法来调用目标方法。
    用method.getName()拿到调用的方法的名字。然后可以利用反射调用目标对象对应的方法。
    例如创建了A1 a1 = new A1()，再用method.invoke(a1,args)可以调用a1对象的同名方法。
  6. args 调用方法时所用的参数。我们把对象交给代理，代理把对象统一交给需要调用的对象。
